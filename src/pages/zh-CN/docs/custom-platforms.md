OmniConsole 内置一组系统平台，你也可以添加自己的：任何可执行文件或应用皆可。一切操作都通过图形界面完成，无需手动编写任何 JSON 配置文件。输入框聚焦时，Windows 11 游戏控制器键盘会自动弹出，无需实体键盘也能填写所有字段。

> ⚠️ **自定义平台为实验性功能。** 内置平台经过测试，但你自行配置的自定义平台则未经测试。配置错误的平台可能无法正常启动，某些情况下还会导致 Xbox 模式 (FSE) 环境不稳定，要恢复可能需要重置应用。首次打开此功能时，OmniConsole 会显示一份须先接受的同意声明。

## 添加平台

在设置界面打开**自定义**选项卡。按下手柄的 **Y 键**，或点击屏幕下方的 **Y 添加** 提示栏，即可打开添加平台的对话框。

## 启动类型

自定义平台采用以下三种启动类型之一：

| 类型 | 说明 |
|---|---|
| 可执行文件 | 指向一个 `.exe` 文件。支持自定义启动参数。内置手柄兼容的文件选择器，无鼠标也能浏览。 |
| Protocol URI | 通过协议链接启动，例如 `steam://`。 |
| 封装应用 | 适用于 MSIX / APPX 应用（UWP 或 Microsoft Store 应用）。以包系列名称搜索已安装的包。 |

## 卡片封面

你可以为自定义平台添加专属的卡片封面图。建议分辨率为 800 × 560 px（10:7 比例）；图片会自动缩放至该尺寸。

## 导入与导出

要分享平台，在卡片上点右键（或长按），选择**导出**，配置便会以 JSON 格式复制到剪贴板。要添加他人分享的平台，使用自定义选项卡的**导入**按钮并粘贴 JSON。
